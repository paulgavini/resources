Practical Investigation Builder - User Guide

This page explains how to use the app and what every button does.

Open The Builder YouTube

1. What this project does

Practical Investigation Builder helps students plan, complete, analyse and report a science practical investigation in one guided workflow.

  • Complete all 12 investigation sections from setup through to conclusion.
  • Build data tables with optional average and standard deviation columns, then create one or more line, scatter or bar graphs.
  • See a live report preview as you work.
  • Save and load work in your browser database (IndexedDB).
  • Export investigations as JSON or Word (.docx), and import JSON backups.
  • Print a clean report (or save as PDF).

2. Quick start

  1. Enter your investigation details in section 1.
  2. Write your research question, hypothesis and variables.
  3. Add risk controls, materials and method steps.
  4. Define data columns and rows, then enter your results.
  5. Choose graph type, x-axis column, and one or more Y-axis columns in the analysis section.
  6. Complete evaluation and conclusion prompts.
  7. Use Save, Export JSON/Word, or Print/PDF when finished.

3. Header controls (top of app)

Starter template (dropdown) Selects one of the built-in investigation examples. Use at the start when you want a scaffolded investigation.
Instructions Opens this guide page in a new tab. Use any time you need help with controls or workflow.
Apply template Loads the selected template into the form fields. After choosing a template from the dropdown.

Export Word Downloads the current investigation as a Word document (.docx). When you need a Word version for editing, marking, or submission.
Import JSON Loads investigation data from a previously exported JSON file. When moving work between devices or restoring a backup.
Print/PDF Opens browser print dialog using print-friendly styling. To print or save the report as PDF.
Preview toggle Hides or shows the live report preview so the editor can use more horizontal space. Use it when you want a wider editing area, then turn the preview back on to check the report layout.
Clear all Clears the current investigation form after confirmation. Only when you want to start a new investigation from scratch.

6. Research question feedback (heuristic validator)

The validator checks if a question is probably specific enough using transparent rules. It is not AI judging.

  • Green: good
  • Amber: needs improvement
  • Red: too vague

The app suggests practical improvements such as naming what you will change and what you will measure.

7. Live report preview and graphs

  • The right-side report preview updates as fields are edited.
  • Each analysis graph card appears in preview when enough valid data is available.
  • You can add extra graph cards and delete any graph after the first one.
  • Each graph can use line, scatter, bar, bubble, pie, doughnut, polar area, or radar display types.
  • Each graph can plot multiple Y-axis columns on the same graph using different colours and a legend.
  • Bubble charts need a separate radius column that is different from the X-axis and Y-axis selections.
  • Each graph also has a checkbox to force compatible axes to start at zero where the selected graph type supports it.
  • Choose trendlines (linear, quadratic, exponential, logarithmic, power, moving average) or set to none.
  • When a trendline is applied, the graph area shows equation, standard deviation, and R² as a percentage below the x-axis label.
  • Trendlines are available for line and scatter graphs, and they are shown when one Y-axis column is selected for that graph.
  • The graph commentary field is for a short summary of what the graph demonstrates before the deeper written analysis.
  • Axis labels include units where units are entered in data column settings.

8. Saving and compatibility notes

  • Saved data is stored in this browser profile using IndexedDB.
  • Saving in one browser/device does not automatically sync to another.
  • Use Export JSON and Import JSON to move data.
  • Running from GitHub Pages works because it is served over HTTP/HTTPS.
  • Direct file:// opening can behave differently in some browsers for module scripts.
