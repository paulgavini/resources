Blockly HTML Builder

Help and Instructions

This page explains how to build HTML and CSS pages in the Blockly editor, how to read the preview, and how to save or load your work.

by Paul Gavini using CODEX

Quick Start

  1. Open the builder and drag one HTML document block into the workspace.
  2. Add content blocks into the document body section.
  3. Add CSS rule blocks into the document head styles section if you want to style the page.
  4. Read the generated code on the right and check the live preview underneath it.
  5. Use Save to download your project file and Load to bring it back later.

What each part does

  • Workspace: the main Blockly area where you drag, connect, and edit blocks.
  • Toolbox: the category list on the left side of the workspace. Click a category to see its blocks.
  • Source code: the exact HTML and CSS being generated from the blocks.
  • Live preview: the rendered page based on the generated code.
  • Status badge: shows whether the workspace is ready or whether you still need an HTML document block.

Buttons across the top

  • Help: opens this instruction page.
  • Save: downloads the current Blockly project as a JSON file.
  • Load: opens a saved Blockly project file.
  • Copy HTML: copies the generated code to the clipboard.
  • Clear workspace: removes all blocks after confirmation.
  • Autosave ready: the app also stores your latest work in local browser storage.

Recommended workflow

  1. Start with Document.
  2. Add headings, paragraphs, sections, links, images, lists, tables, or forms.
  3. Use the Graphics and Media categories for pictures, SVG, audio, and video.
  4. Add CSS rules only after the page structure is working.
  5. Check the source code and live preview often while building.

Toolbox categories

  • Document: the root page block and comments.
  • Text: headings and paragraphs.
  • Structure: page sections such as header, nav, main, section, div, and footer.
  • Graphics: links, images, figures, responsive pictures, and SVG drawing blocks.
  • Media: audio, video, and source blocks.
  • Lists, Tables, Forms: common classroom HTML structures.
  • CSS Rules and CSS Properties: beginner styling blocks that go into the document head.

Using the divider

You can resize the border between the workspace and the preview column.

  • Drag the vertical divider left or right to change the width of the preview area.
  • Click the divider and use ArrowLeft or ArrowRight to resize with the keyboard.
  • Use Home for the smallest preview width and End for the largest.
  • The app remembers the preview width in your browser.

Saving and loading

Your project file contains the Blockly workspace, not just the HTML output.

{
  "version": 1,
  "format": "blockly-html",
  "workspace": {}
}

That means you can reopen the file later and continue editing the same blocks.

Important tips

  • Hover over blocks to read their help text and examples.
  • Many blocks have attributes on the second line, such as id, class, title, width, or target.
  • Dropdowns are used where possible to keep the project beginner friendly.
  • The generated code will usually stay cleaner if you leave optional fields blank unless you need them.

Current limits

  • This project focuses on HTML and CSS only.
  • JavaScript and script tags are intentionally out of scope in this stage.
  • canvas is not included yet because useful canvas work depends on JavaScript.
If the workspace says Add document, place one HTML document block first. If it says One document only, remove any extra document blocks.

Example build order

  1. Add an HTML document block.
  2. Place an h1 and a paragraph in the body.
  3. Add an image or figure block.
  4. Add a CSS rule for body or h1 in the head styles section.
  5. Check the preview, then save the project.
