Mathematics Investigation Builder - User Guide

Use this guide to learn the app layout, the writing expectations, and the export options before starting your investigation.

1. What this project does

Mathematics Investigation Builder helps students in Years 7 to 12 plan, draft, and export a mathematics investigation report from start to finish.

  • Build the report in the order: setup, introduction, investigation, data, graphs, analysis, conclusion, export.
  • If this is your first time using the app, work from top to bottom and use Apply template to start with an example.
  • Use a Tabulator data table and Chart.js graphs from local vendor files inside math_inv/vendor.
  • See a live continuous-prose report preview as you work, and use the preview icon to hide or re-enable it when you need more editing space.
  • Work autosaves to the browser database every 15 seconds and loads automatically when the app opens or refreshes.
  • Export the report as JSON, Word, or PDF. Word exports use Calibri 10 with 1.27 cm page margins.

Save icon Saves immediately instead of waiting for the next autosave cycle.

Every icon button in the toolbar also has a tooltip. Hover over an icon if you are unsure what it does.

3. Report Structure Guide

Report part Tense What it must do
Introduction Future tense Hook the reader, explain the problem, identify the field of mathematics, describe that field, and connect the mathematics to the problem.
Mathematical investigation Present tense Introduce each piece of mathematics, complete the mathematics, then explain what it shows. Repeat this cycle until the investigation is complete.
Analysis Past tense Analyse patterns, written evidence, assumptions, reasonableness, strengths, limitations, and improvements.
Conclusion Past tense Link back to the introduction, restate the major findings, and clearly state whether the problem was solved.
A strong mathematics report should read as continuous prose. Use the writing fields to explain the mathematics, and use tables and graphs as supporting evidence rather than the whole report.

4. How the app maps to the report

App section Purpose
Investigation setup Stores the title, class details, task type, and the problem focus.
Introduction Captures the hook, problem context, field of mathematics, explanation of the field, and the connection to the task.
Mathematical investigation Uses repeatable investigation cycles so each stage can be introduced, shown, and explained.
Data tables Builds and edits the main data table with local Tabulator.
Graphs Builds one or more graphs from the table and records what the graphs show.
Analysis Collects the pattern of results, evidence, assumptions, reasonableness, strengths, limitations, and improvements.
Conclusion Combines the retrospective link back, major findings, and final solution statement.

5. Data and graph tools

Data tables

  • Edit the column settings first, then drag the column rows when you want to change the order of the data-entry headings.
  • Use Add column or Add trial column underneath the column list when needed.
  • The data table starts with 3 rows, and you can use Add row underneath the table when more rows are needed.
  • Edit values directly inside the Tabulator table.
  • Use the average and standard deviation checkboxes at the top of the data section when those summary columns help the investigation.

Graphs

  • Add one or more graph cards.
  • Select the X-axis column and one or more Y-axis columns.
  • Choose from all built-in Chart.js graph types: line, scatter, bar, bubble, pie, doughnut, polar area, and radar.
  • Use a separate radius column for bubble charts.
  • Use a trendline when one Y-axis series is selected on a line or scatter graph.
  • Use the graph commentary box to state what the graph demonstrates before the deeper analysis.

6. Writing reminders

Do

  • Describe exactly what you did in the investigation.
  • Let the language of mathematics speak through the workings.
  • Explain what each mathematical step, table, or graph showed.
  • Use paragraph-style analysis for strengths, limitations, and improvements.

Do not

  • Write the report like a task sheet summary.
  • Explain basic procedures such as "multiply the length by the width" instead of just doing the mathematics.
  • Leave the conclusion as a repeat of the introduction without a solved/not solved statement.
Mathematical notation input: type x^2 for a superscript and x_1 for a subscript. For longer exponents or indices, use grouped forms such as x^{n+1} or x_(i+1). These forms render automatically in the live report preview and the Word export.

8. Quick workflow

  1. Complete the setup details.
  2. Write the introduction in future tense.
  3. Add one or more mathematical investigation cycles in present tense.
  4. Enter the data and build the graphs.
  5. Write the analysis in past tense.
  6. Finish the conclusion in past tense.
  7. Check the preview, then export as Word or PDF. The Word export is formatted with Calibri 10 and 1.27 cm margins.
